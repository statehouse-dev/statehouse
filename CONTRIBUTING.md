# Contributing to Statehouse

Thank you for your interest in Statehouse!

## Important Note

Statehouse is **licensed proprietary software**, not open source.

At this time, we are **not accepting external contributions** during the MVP development phase.

## Future Contribution Model

Once the MVP is stable, we plan to:
- Accept bug reports and feature requests via GitHub Issues
- Consider accepting patches and improvements under a Contributor License Agreement (CLA)
- Provide clear contribution guidelines for approved contributions

## Getting Help

If you're evaluating Statehouse for your use case:
- Check the [documentation](docs/)
- Open a GitHub Discussion for questions
- Email: [support@statehouse.dev](mailto:support@statehouse.dev)

## Development Practices

This project employs AI-assisted development practices. See [docs/dev/ai_assisted_dev.md](docs/dev/ai_assisted_dev.md) for details on our workflow and quality assurance approach.

## Security Issues

Please see [SECURITY.md](SECURITY.md) for how to report security vulnerabilities.
